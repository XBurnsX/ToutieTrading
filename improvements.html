01
MARKET CONDITION FILTER
CRITIQUE ▶
Ton edge vient pas juste de "quand entrer" — il vient surtout de quand NE PAS trader. Module global qui filtre avant même que la Strategy évalue.
📉
Volatilité trop basse → skip // ATR en dessous du seuil configurable
📈
Volatilité trop haute → skip // ATR au-dessus du seuil configurable
💸
Spread trop élevé → skip // spread > X pips configurable par paire
🌙
Session morte → skip // ex: session asiatique pour paires EUR/USD
📰
News importantes → skip // voir module News #11
→ MODULE GLOBAL — s'applique AVANT toute Strategy. Configurable par paire dans Settings.
02
RISK ADAPTATIF
TRÈS IMPORTANT ▶
Le RiskPercent fixe c'est V1. En V2, le risk s'adapte automatiquement selon la performance récente.
📉
X pertes consécutives → réduire risk // ex: 3 pertes → risk / 2
📈
Equity en hausse → augmenter légèrement // +X% equity → risk * 1.25, plafonné
🔻
Drawdown actif → couper risk // DD > Y% → risk minimum jusqu'à récupération
⚙️
Tous les seuils configurables dans Settings de la Strategy
→ Extension de RiskEngine — V2

04
SCORE GLOBAL DE STRATEGY
TRÈS IMPORTANT ▶
Ne jamais classer par profit seul — ça sort des strategies chanceuses. Le score pondéré évite les combinaisons "overfittées".
💰
Net Profit ×0.4
📉
1 / Max Drawdown ×0.3 // DD faible = meilleur score
⚖️
Profit Factor ×0.2
🔢
Nombre de trades ×0.1 // pénalité si < 30 trades
Score = Profit × 0.4
+ (1/DD) × 0.3
+ PF × 0.2
+ Trades × 0.1

11
NORMALISATION PAR PAIRE (ATR)
CRITIQUE ▶
Chaque paire est différente. EURUSD ≠ GBPJPY ≠ NASDAQ. Des pips fixes cassent ta strategy selon le marché. Tout doit être basé sur l'ATR de la paire.
📏
SL basé sur ATR → ex: SL = ATR × 1.5 au lieu de X pips fixes
🎯
TP basé sur ATR → ex: TP = ATR × 3.0 (ratio 2:1 conservé mais adapté)
🔍
Filtres basés sur ATR → ex: spread trop élevé = spread > ATR × 0.1
⚙️
Multiplicateurs ATR configurables dans Settings par Strategy
✅
Une même Strategy tourne correctement sur EURUSD, GBPJPY et indices sans ajustement manuel
→ Extension de RiskEngine + StopLossRule — ATR déjà calculable depuis IndicatorEngine
12
COOLDOWN ENTRE TRADES
TRÈS IMPORTANT ▶
Évite l'overtrading et le revenge trading version bot. Après un trade, le bot attend avant de chercher une nouvelle entrée.
⏱️

🏗️

🔴
Cooldown renforcé après perte → X bougies supplémentaires après un trade perdant
⚙️
Mode cooldown configurable dans Settings : bougies fixes ou structure
→ Extension de TradeManager — configurable par Strategy via Settings
13
FTMO SIMULATOR
MODULE SÉPARÉ ▶
Simuler exactement les règles FTMO en replay et live-démo. Savoir si la strategy aurait passé un challenge sans violer les objectifs — avant de risquer de l'argent réel.
PROFIL FTMO 2-STEP STANDARD
🎯
Phase 1 (Challenge) : +10% profit, max 5% daily loss, max 10% total loss, min 4 jours trading
✅
Phase 2 (Verification) : +5% profit, mêmes règles de loss, min 4 jours trading
🌍
Timezone FTMO : Europe/Prague (CE(S)T) — reset journalier à minuit Prague
💸
Inclut : floating P/L + commissions + swaps dans tous les calculs
MOTEURS FTMO
📉
FtmoDailyLossTracker : reset à minuit Prague. Limite = balance_at_midnight - 5% capital initial. Violation si equity descend sous cette limite (closed P/L + floating + commissions + swaps).
🔻
FtmoMaxLossTracker : limite globale = initial_capital - 10%. Violation à n'importe quel moment, positions ouvertes incluses.
📅
FtmoTradingDaysCounter : min 4 jours. Un jour = au moins une position initiée. Si trade multi-jours → seul le jour d'ouverture compte. Jours pas forcément consécutifs.
⚙️
FtmoPhaseEngine : CurrentPhase = Challenge | Verification | Passed | Failed. Validation seulement si toutes les positions sont fermées.
RULE BREACH SYSTEM
🚨
FtmoBreachType : None | DailyLoss | MaxLoss — quand cassé : challenge = Failed, stop nouveaux trades, bannière rouge UI, log complet
📝
Log breach : FtmoBreachTimestamp, FtmoBreachEquity, FtmoBreachReason
UI — PANNEAU FTMO (REPLAY + LIVE)
📊
Phase actuelle | Profit actuel | Target restant | Daily Loss restant | Max Loss restant | Trading Days validés
⚠️
Warning avant trade : si le prochain trade pourrait casser FTMO → alerte visible
🔴
Status : Running | Passed | Failed — bannière rouge si Failed
DB — CHAMPS LOGGÉS PAR TRADE
💾
ftmo_phase ftmo_daily_loss_remaining ftmo_max_loss_remaining ftmo_trading_days ftmo_status_after_trade ftmo_breach_type
MODE SÉCURITÉ FTMO
🛡️
FtmoSafeMode toggle : stop journée AVANT la vraie limite FTMO (ex: stop à -3% au lieu de -5%)
📉
Réduire risk après pertes consécutives
🚫
Refuser trades si marge FTMO restante trop faible
⚙️
Seuils SafeMode configurables dans Settings
→ Module séparé utilisant le même moteur replay/live. Même timezone FTMO Prague gérée indépendamment de l'heure Québec du bot.
